\section{Conclusion}
\label{sec:conclusion}

This project produced a credible lossless compression tool and a clear comparison between three different design choices. In the final benchmark set, \texttt{v9} achieved the best compression ratio, \texttt{v8} was the fastest version, and \texttt{v5} remained the best overall balance. It combined good compression performance with much lower cost than \texttt{v9} and much better output size than \texttt{v8}.

A main lesson from the project is that compression performance did not come from one isolated algorithmic idea, but from how preprocessing, modeling, and implementation details worked together. BWT and MTF were useful because they made the statistical model more effective, while choices such as block handling, escape estimation, and candidate selection also had a strong impact. The comparison with \texttt{v8} also made the trade-off very clear: higher speed is possible, but usually at the cost of weaker compression.

The most significant theoretical insight came from file D. By identifying it as glibc \texttt{rand()} output and compressing 2\,MB to 14 bytes, \texttt{v10} demonstrated that Shannon entropy and Kolmogorov complexity are fundamentally different measures of information content. Standard compressors are bounded by Shannon entropy because they model statistical distributions. The PRNG detection module instead models the \emph{computational process} that generated the data, achieving compression that no statistical method can match. This illustrates a core principle of Algorithmic Information Theory: the true information content of data is determined by the shortest program that produces it, not by its byte frequency distribution.

Future work can follow three directions. The statistical line can be improved by reducing the cost of broader search and handling difficult binary files better. The speed-oriented \texttt{v8} line can continue as a separate compressor for throughput-sensitive cases. And the Kolmogorov compression approach in \texttt{v10} can be extended to detect other PRNG families (Mersenne Twister, xorshift, LCG variants) or other algorithmically generated data, broadening the class of high-entropy inputs that can be compressed.
